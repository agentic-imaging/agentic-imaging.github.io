Accessibility

The AgX project is committed, in good faith, to making this site usable by the widest possible audience, including people with disabilities. This commitment is informed by the digital-accessibility policies of Caltech, MIT, and HKU. We aim to meet the W3C Web Content Accessibility Guidelines (WCAG) 2.1, Level AA.

What we do

  • Semantic landmarks (header, nav, main, article, footer), a logical heading order, and a skip-to-content link on every page.
  • Full keyboard operability — navigation, theme toggle, the table-of-contents drawer (Esc to close), and copy buttons — with visible :focus-visible indicators.
  • Text and interface colors chosen for AA contrast in both light and dark themes; no information is conveyed by color alone.
  • Text alternatives for the diagram (an SVG with role="img", a title, and a long description).
  • Responsive, zoomable layouts down to small phone widths, with no horizontal scrolling of body text and generously sized touch targets.
  • prefers-reduced-motion is honored, and the site is fully readable without JavaScript — scripting only adds conveniences.
  • A self-hosted spoken audio narration of the main page and the tour — start it from the speaker button in the top corner — with a live, synced single-line caption, as an audio (and captioned-text) alternative to the visual content.

Known limitations

  • Some data figures and the interactive demo convey information visually; screen-reader users get the same content linearly, but text alternatives for data figures are limited.
  • Without JavaScript the small-screen table-of-contents drawer is unavailable (all headings and the in-page order remain navigable).
  • The site has not had an independent third-party audit, so full WCAG 2.1 AA conformance is not certified — this is a good-faith, self-assessed statement.

Feedback

If you encounter a barrier, please email agentimaging@gmail.com and we will respond promptly.
