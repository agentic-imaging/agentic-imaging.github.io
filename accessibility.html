Accessibility

This site targets WCAG 2.2 level AA. Implemented measures:

  • Semantic landmarks (header, nav, main, article, footer) and a skip-to-content link on every page.
  • Full keyboard operability: navigation, theme toggle, table-of-contents drawer (with Esc to close), collapsible sections, and copy buttons are all reachable and operable by keyboard, with visible :focus-visible indicators.
  • Color contrast at or above AA in both light and dark themes, for text, links, chips, and the sample-evidence banner.
  • Alternative text for the diagram (an SVG with role="img", a title, and a long description) and no meaning conveyed by color alone.
  • prefers-reduced-motion is honored: smooth scrolling and disclosure animations are disabled when reduced motion is requested.
  • The site is fully readable without JavaScript; scripting only adds conveniences (theme toggle, scroll-spy highlight, copy buttons, sample-table rendering — a noscript note points at the raw sample file).
  • Responsive layout down to small phone widths, with no horizontal scrolling of body text and touch targets sized for WCAG 2.2 (target-size minimum).

Known limitations

  • Without JavaScript the table of contents is unavailable on small screens (headings and the in-page order remain fully navigable).
  • The interactive trace demo presents code excerpts in horizontally scrollable blocks; screen-reader users get the same content linearly.

Accessibility feedback is welcome — a contact route ships with the public release (this preview intentionally publishes no contact addresses).
